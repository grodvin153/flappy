2 - Control de movimiento

Movimiento de las tuberías

Para mover las elementos tuberías, vamos a crear una variable SPEED a la que asignaremos un valor 3 (posición que cambiará la tubería de forma horizontal con cada ejecución del programa). Cuanto mayor sea este valor, más difícil será el juego.

ver código

SPEED = 3                          # se crea una variable SPEED para la velocidad del juego con valor 3

A continuación, vamos a definir una función update_pipes que usaremos para actualizar la posición de las tuberías.

Dentro de esta función:

  • Reduciremos la posición horizontal izquierda (left) de la tubería superior pipe_top según el valor indicado en la variable de velocidad SPEED.
  • Reduciremos la posición horizontal izquierda (left) de la tubería inferior pipe_bottom según el valor indicado en la variable de velocidad SPEED.
  • Si el lado derecho (right) de la tubería superior pipe_top es inferior a 0, significa que la tubería ha desaparecido por el lado izquierdo de la pantalla (podríamos usar igualmente la tubería inferior, ya que ambas tienen la misma posición horizontal).
    • Se reinician las posiciones de las tuberías llamando a la función reset_pipes creada anteriormente.

ver código

def update_pipes():
    pipe_top.left = pipe_top.left - SPEED         # se mueve la tubería superior con la velocidad SPEED
    pipe_bottom.left = pipe_bottom.left - SPEED   # se mueve la tubería inferior con la velocidad SPEED
    if pipe_top.right < 0:            # Cuando la tubería supera el borde izquierdo se vuelve a reiniciar la posición en la derecha
        reset_pipes()

También se puede utilizar esta otra forma de reducir la posición de pipe_top y pipe_bottom

def update_pipes():
    pipe_top.left -= SPEED
    pipe_bottom.left -= SPEED
    if pipe_top.right < 0:
        reset_pipes()

Sentencia condicional

Observa que en el código anterior hemos utilizado una sentencia condicional if-else. Una sentencia if consiste en:

  • La palabra reservada if , da inicio al condicional if.
  • La siguiente parte es la condición. Esta puede evaluar si la declaración es verdadera o falsa. En Python estas son definidas por las palabras reservadas (True or False). 
  • Paréntesis (()) Los paréntesis son opcionales, no obstante, ayudan a mejorar la legibilidad del código cuando más de una condición está presente.
    Dos puntos : cuya función es separar la condición de la declaración de ejecución siguiente.
  • Una nueva línea. Un nivel de indentación de cuatro espacios, que es una convención en Python. El nivel de indentación es asociado con la estructura de la declaración que sigue.
  • Finalmente, la estructura de la sentencia. Este es el código que será ejecutado, únicamente si la sentencia a ser evaluada es verdadera. Es posible tener múltiples líneas en la estructura de código que pueden ser ejecutadas; en este caso es necesario tener cautela en cuanto a que todas las líneas tengan el mismo nivel de indentación.
if condicion:
    ejecutar codigo si la condicion es True
else:
     ejecutar codigo si la condicion es False

Cuando la expresión if se evalúa como True, entonces ejecuta el código que le sigue. Pero si se evalúa como False, entonces ejecuta el código que sigue después de la sentencia else .

La sentencia else está escrito en una nueva línea, posterior a la última línea del código desplazado a la derecha y en la misma posición de la condición if. Una sentencia else siempre tiene como prerrequisito una sentencia if, siendo a la vez, parte de él.

El código posterior a la sentencia condicional else se ejecuta sí y solo sí la primera parte, del condicional if se evalúa como False. Si tu sentencia en la condicional if se evalúa como True, será ese bloque de código el que se ejecutará y el código que sigue a else no se ejecutará nunca.

Movimiento del pájaro

En primer lugar vamos a definir el estado inicial del personaje bird:

  • No está muerto, por lo que el atributo dead tiene valor false (esta información la utilizaremos más adelante).
  • La velocidad vertical (vy) tiene valor 0. Este valor lo utilizaremos más adelante para definir la gravedad que tendrá el pájaro en su caída al suelo.

ver código

bird.dead = False
bird.vy = 0

Creamos una variable para la definición de la gravedad GRAVITY con valor 0.3. Al igual que con la variable SPEED, a mayor valor de gravedad, más dificultad tendrá el juego.

ver código

GRAVITY = 0.3

A continuación vamos a definir una función update_bird para manejar el movimiento del pájaro:

  • El pájaro tiene una velocidad vertical (vy) del valor definido en GRAVITY.
  • La posición vertical y del pájaro se ve incrementa según la velocidad vertical vy. Al tener un valor positivo, la posición vertical es cada vez mayor, por lo que significa que el pájaro cae.
  • La posición horizontal del pájaro x se mantiene en 75.

ver código

def update_bird():
    bird.vy += GRAVITY      # asignamos la velocidad vertical del pájaro al valor de la gravedad
    bird.y += bird.vy       # la posición vertical del pájaro aumenta (cae) según el valor de la velocidad vertical
    bird.x = 75             # la posición horizontal del pájaro se mantiene a 75

Actualización de la posición de tuberías y pájaro

Una vez definidas las funciones update_pipe y update_bird, éstas deben ser llamadas de forma continua durante toda la ejecución del juego.

Para ello, vamos a definir y utilizar otra de las funciones reservadas de Pygame para la actualización del juego llamada update.

Dentro de esta función se llama a las funciones update_pipe y update_bird.

ver código

def update():              # procedimiento estándar de actualización
   update_pipes()               # se actualizan las tuberías
   update_bird()                # se actualiza el actor "bird"
